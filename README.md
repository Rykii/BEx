## 🎯 核心概念

| 概念 | 说明 |
|------|------|
| **FICC** | 固定收益(Fixed Income)、货币(Currency)、商品(Commodities)的统称 |

### Agent职责

| Agent | 职责 | 核心能力 |
|-------|------|----------|

## 🔗 调用关系

### Agent间调用逻辑

```
BEx (主控)
   │
   ├─▶ Recon (输入: 资产类型 + 探查范围)
   │       │
   │       └─▶ 输出: 权威信源清单 + URL列表
   │
   ├─▶ Archivist (输入: URL列表)
   │       │
   │       └─▶ 输出: Markdown知识库文件
   │
   └─▶ Analyst (输入: 知识库文件 + 可选模板)
           │
           └─▶ 输出: 综合探查报告
```

### 数据传递

| 传递方向 | 传递内容 | 格式 |
|----------|----------|------|
| BEx → Recon | 资产类型、探查范围 | 自然语言 |
| Recon → Archivist | URL清单、文件类型 | JSON/YAML |
| Archivist → Analyst | Markdown文件路径 | 文件路径列表 |
| Analyst → BEx | 综合探查报告 | Markdown文件 |

## 📖 使用方法

### 方式一：直接对话（推荐）

向BEx发送任务请求：

```
我需要了解【外汇NDF】的【监管框架和交易规则】
```

BEx会自动：
1. 解析任务（资产类型=外汇NDF，探查范围=监管框架+交易规则）
2. 调用Recon获取信源
3. 调用Archivist采集数据
4. 调用Analyst生成报告

### 方式二：指定探查范围

```
资产类型：碳配额回购
探查范围：全国碳市场碳配额交易回购业务规则、中国人民银行相关指引
```

### 方式三：批量调研

```
帮我调研以下资产类型的官方规则：
1. 债券买断式回购
2. 商业银行CDS业务
3. 权益衍生品中期权
```

### 典型对话示例

```
用户：外汇NDF的监管框架是什么？
     BEx：
     1. 确认理解（资产类型：外汇NDF，探查范围：监管框架）
     2. 调用Recon → 生成信源清单（SAFE/CFETS/NDF离岸规则）
     3. 调用Archivist → 采集相关法规
     4. 调用Analyst → 生成探查文档
     5. 输出：包含监管框架、交易规则、待核实事项的Markdown
```

```markdown

title: 碳配额回购交易规则

  - domain: ceex.cn
    url: https://ceex.cn/rules/xxx
    date: 2025-12-15
    version: "2.0"

last_updated: 2026-05-19
---

# 碳配额回购交易规则

## 📌 政策概览
...

## 📋 交易规则
### 基本要素
...

## ⚠️ 风险提示
- 【待核实】：需要核实的内容

## 📚 参考资料
1. [规则原文](https://ceex.cn/...)
```

### 文档结构

每个探查文档包含：

| 章节 | 内容 |
|------|------|
| YAML Front Matter | 资产类型、版本、源文件清单、状态 |
| 政策概览 | 核心监管框架解读 |
| 交易规则 | 基本要素、期限、报价方式 |
| 清结算规则 | 清算机构、保证金要求 |
| 风险管理 | 风控要求、限制条件 |
| 【待核实】标注 | 未找到明确依据的内容 |
| 参考资料索引 | 官方源链接 |

## ✨ 特性

- **源追溯**：所有结论标注【依据】，指向官方文件具体条款
- **版本优先**：2024-2026年有效版本 > 更早版本
- **冲突处理**：新旧版本差异时，以最新且未废止版本为准
- **禁止编造**：未找到明确依据时标注【待核实】
- **客观性**：不补充个人解释，保持原始监管语义
